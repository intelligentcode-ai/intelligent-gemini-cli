# Documentation Patterns

**PURPOSE:** Shared patterns for all documentation generation

## Template Structure

```markdown
# [TITLE]
## Problem
[What/Why/When]
## Solution  
[How/Steps]
## Examples
[Code/Usage]
## Validation
[Success criteria]
```

## Quality Standards
- Clear problem statement
- Actionable steps
- Concrete examples
- Measurable success

## Metadata Format
```yaml
---
date: YYYY-MM-DD
version: X.Y.Z
category: [category]
---
```
